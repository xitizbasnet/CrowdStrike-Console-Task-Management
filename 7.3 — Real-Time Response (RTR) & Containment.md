## 🎯 Objective

Use **CrowdStrike Falcon Real-Time Response (RTR)** to investigate, contain, and remediate a potentially compromised endpoint.

---

# 📋 Procedure

## ⚡ Step 1: Initiate RTR Session

> **Purpose:** Establish live remote access to an endpoint for investigation and remediation.

### Launch RTR Session

Navigate to:

```text id="rtr1"
Response → Real Time Response → Hosts → Select Host → Start Session
```

---

### RTR Session Behavior

* RTR opens a **terminal-like interface** directly to the endpoint
* No VPN is required
* All commands executed in RTR are fully **audited in Falcon**
* Sessions remain active even if the host is **network contained**

---

### Multi-Analyst Access

* Multiple analysts can view the session
* Only one analyst can execute commands at a time

---

## 🧰 Step 2: Common RTR Investigation Commands

> **Purpose:** Collect forensic and runtime evidence from the endpoint.

### Process and System Inspection

```bash id="rtr2"
ps
```

* Lists running processes

---

```bash id="rtr3"
netstat
```

* Displays active network connections

---

### Persistence Investigation

```bash id="rtr4"
reg query HKLM\Software\Microsoft\Windows\CurrentVersion\Run
```

* Checks registry-based persistence mechanisms

---

### File System Review

```bash id="rtr5"
ls C:\Users\Public\
```

* Checks common malware drop locations

---

### User Activity Review

```bash id="rtr6"
history
```

* Displays command history for the suspect session

---

### File Collection

```bash id="rtr7"
get C:\Temp\suspicious.exe
```

* Downloads suspicious files for offline analysis

---

## 🛠️ Step 3: Remediation via RTR

> **Purpose:** Contain and remediate malicious activity directly from the endpoint.

### Process Termination

```bash id="rtr8"
kill
```

* Terminates malicious processes

---

### File Removal

```bash id="rtr9"
rm C:\Temp\malware.exe
```

* Deletes malicious files from the endpoint

---

### Run Remediation Scripts

```bash id="rtr10"
runscript -CloudFile=cleanup_ransomware.ps1
```

* Executes predefined remediation workflows

---

### File Deployment

```bash id="rtr11"
put quarantine_tool.exe
```

* Pushes remediation tools from Falcon cloud to endpoint

---

### Antivirus Scan Execution

```bash id="rtr12"
runscript -CloudFile=av_scan.ps1
```

* Triggers full system antivirus scan

---

## 🔍 Step 4: Post-Containment Verification

> **Purpose:** Confirm that the system is clean before restoring normal operations.

### Verification Checks

* Confirm no malicious processes are running:

```bash id="rtr13"
ps | grep malware_name
```

* Validate persistence mechanisms are removed:

  * Registry keys
  * Scheduled tasks
  * Services

* Verify no active C2 connections:

```bash id="rtr14"
netstat
```

---

### Falcon Verification

* Trigger an on-demand scan via policy

---

### Release Containment

* Only after verification:

  * **Actions → Lift Network Containment**

---

### Post-Remediation Monitoring

* Monitor the host for:

  * 24–48 hours after remediation

---

# 📊 Best Practice Tips

### 📌 Pre-Build Response Scripts

* Store remediation scripts in Falcon cloud:

  * Response → Scripts

> Enables rapid deployment during incidents

---

### 📌 Exercise Extreme Caution with RTR

* RTR scripts run as **SYSTEM**
* Destructive commands can cause irreversible damage

---

### 📌 Use One-Liner PowerShell Carefully

```bash
runscript -Raw
```

* Enables inline PowerShell execution without script staging

---

### 📌 Contain First, Investigate Second

* Always isolate the endpoint before deep investigation
* Prevents further spread or data loss

---

### 📌 Never Release Containment Prematurely

* Do not rely on assumptions
* Always verify using RTR commands and clean scan results
