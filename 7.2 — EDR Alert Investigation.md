## 🎯 Objective

Investigate a **CrowdStrike Falcon detection** from initial alert through analysis, verdict, and response actions.

---

# 📋 Procedure

## 🚨 Step 1: Review Detection Alert

> **Purpose:** Understand the context, severity, and scope of the detection.

### Navigate to Detection

* Go to:

  * **Endpoint Security → Detections**
* Select the relevant detection.

---

### Review Key Fields

Analyze the following attributes:

* Severity level
* MITRE ATT&CK:

  * Tactic
  * Technique
* Affected host
* Associated user
* Triggering process

---

### Process Tree Analysis

* Review full **process chain**
* Identify:

  * Parent process
  * Child processes
  * Execution flow leading to detection

---

### Timeline Review

* Examine events:

  * Before detection
  * During detection
  * After detection

---

### Context Validation

Check:

* Is the host a **high-value asset**?
* Was the user expected to be active (on shift)?

---

## 🔍 Step 2: Investigate in Falcon Insight (Threat Hunting)

> **Purpose:** Perform deeper threat hunting using event-level telemetry.

### Navigate to Investigation Tool

* Go to:

  * **Investigate → Falcon Insight → Event Search**

---

### Process Execution Query

```text id="edr1"
event_simpleName=ProcessRollup2 FileName=powershell.exe
```

---

### Time-Based Filtering

```text id="edr2"
aid=<host_id> timestamp>last24h
```

---

### Pivot Analysis

Investigate relationships using:

* File hashes
* Parent processes
* Network connections

---

### Network Activity Query

```text id="edr3"
event_simpleName=NetworkConnectIP4 LocalAddressIP4=10.1.1.100
```

---

## 🧠 Step 3: Analyse Process Graph

> **Purpose:** Visually reconstruct attack behavior and execution flow.

### Access Process Graph

* Navigate:

  * Detection → **Process Graph Tab**

---

### Key Analysis Points

Identify:

* Unusual parent-child relationships

  * Example: Word → PowerShell (possible macro execution)

---

### Command Line Inspection

Check for:

* Encoded commands
* Download cradles
* Obfuscated execution patterns

---

### File Activity Review

Investigate:

* Files created
* Files modified
* Files dropped by suspicious processes

---

### Network Activity Review

Determine:

* Outbound connections
* Destination IP addresses
* Suspicious external communication

---

## ⚖️ Step 4: Verdict & Action

> **Purpose:** Classify the detection and execute appropriate response actions.

---

### True Positive Response

If confirmed malicious:

* Set detection status to:

  * **In Progress**
* Assign to:

  * **L2 Analyst**

---

### Containment Actions

* Isolate host:

  * **Actions → Contain**
* Note:

  * Endpoint remains able to communicate with Falcon cloud for management

---

### Evidence Collection

* Use:

  * **Real Time Response (RTR)**
* Download suspicious files for analysis

---

### False Positive Response

If benign:

* Set status:

  * **False Positive**
* Add exclusion rule if appropriate
* Document reasoning in detection comments

---

# 📊 Best Practice Tips

### 📌 Always Inspect Full Command Line

* Do not rely on truncated views
* Full arguments may reveal:

  * Encoded payloads
  * Execution parameters

---

### 📌 Validate Hash Reputation

Check file hashes using:

* VirusTotal
* Falcon Intelligence

> Never rely on a single source when determining false positives.

---

### 📌 Understand Containment Limitations

* Network containment does **NOT** prevent:

  * Lateral movement via domain trust

> Always disable compromised Active Directory accounts when needed.

---

### 📌 Preserve Evidence Before Remediation

* Export:

  * Process graphs
  * Event timelines

Before taking containment or cleanup actions.

---

### 📌 Create Detection Improvements

* Convert confirmed true positives into:

  * Custom IOA rules

> This helps detect future variants not covered by machine learning signatures.
