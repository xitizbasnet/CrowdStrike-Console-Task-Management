## 💻 Step 2: Host Management

> **Purpose:** Deploy and verify Falcon sensor installation across endpoints.

### Sensor Download

Navigate to:

```text id="fc1"
Hosts → Sensor Downloads
```

* Download deployment packages for:

  * Windows
  * Linux
  * macOS

---

### Windows Deployment Command

```cmd id="fc2"
msiexec /i falcon-sensor.msi CID=<CID> /quiet
```

---

### Linux Deployment Commands

```bash id="fc3"
sudo rpm -ivh falcon-sensor-<version>.el8.x86_64.rpm
sudo /opt/CrowdStrike/falconctl -s --cid=<CID>
```

---

### Verification

Check sensor status:

* Navigate to:

  * **Hosts → All Hosts**
* Filter:

  * **Last Seen < 1 hour**

---

## 🛡️ Step 3: Configure Prevention Policies

> **Purpose:** Define endpoint protection rules using Falcon prevention capabilities.

### Create Policy

Navigate to:

```text id="fc4"
Configuration → Prevention Policies → Create New Policy
```

---

### Enable Protection Features

Activate the following:

* Machine Learning Protection:

  * Moderate+ (Production)
  * Aggressive (High-risk systems)

* Exploit Mitigation

* Ransomware Protection

* Suspicious Process Detection

* Custom IOA (Indicators of Attack) Rules

---

### Policy Assignment

Assign policies to host groups based on OS:

* Windows Servers
* Windows Workstations
* Linux Systems

# 📊 Best Practice Tips

### 📌 Use Enterprise Deployment Tools

Deploy Falcon sensors using:

* SCCM
* Microsoft Intune
* Ansible

> Avoid manual installation at scale.

---

### 📌 Pilot Before Full Rollout

* Test policy changes on a **5–10% pilot group**
* Validate before enterprise-wide deployment

---

### 📌 Monitor Sensor Health

* Identify **Reduced Functionality Mode (RFM)** sensors
* These sensors are not fully protected

> Check sensor status daily.

---

### 📌 Enable Kernel-Level Visibility

* Enable kernel-level sensor mode on critical servers
* Provides deepest system visibility and detection capability

---

### 📌 Automate Sensor Monitoring

* Use API-based monitoring
* Detect:

  * Stale sensors
  * Offline endpoints
